# DuaSaku Agent Guide

- Start by reading `PROGRESS.md`, `.agent/ARCHITECTURE.md`, and the files you will change.
- This is an Expo + React Native app with Expo Router: `index.js` boots `expo-router/entry`, routes live in `app/`, shared logic in `src/`, and native Android code in `android/`.
- `app/_layout.tsx` is the app bootstrap: it loads fonts, initializes i18n, fetches the Supabase session, registers notifications/background tasks, and gates access behind biometric unlock when enabled.
- `app/(tabs)/_layout.tsx` owns the global tab shell, floating `+` action, and `SmartInputSheet` bottom sheet.
- Data flow is screen-to-service, not screen-to-screen: `SmartInputSheet.tsx` parses free text, inserts a transaction into Supabase, then emits `transaction_added`; `app/(tabs)/index.tsx` listens and refreshes.
- Keep auth/session tokens in SecureStore via `src/lib/supabase.ts`; use `AsyncStorage` only for UI preferences in `src/store/useUserStore.ts`.
- Localization keys live in `src/lib/i18n.ts`; add both `en` and `id` entries when introducing new user-facing text.
- External integrations to preserve: Supabase, `expo-notifications`, `expo-local-authentication`, `expo-location`, background fetch, and `@gorhom/bottom-sheet`.
- Android config is intentionally prebuilt: `android/settings.gradle`, `android/app/build.gradle`, and `MainApplication.kt`/`MainActivity.kt` include Expo and RN autolinking, Hermes, and New Architecture wiring.
- Do not edit generated Expo blocks in Kotlin files; preserve `// @generated ... DO NOT MODIFY` sections and the splash-screen registration in `MainActivity.kt`.
- Preserve the React Native/Expo build chain: `babel.config.js` keeps `react-native-reanimated/plugin` last, and `app.json` enables `newArchEnabled`, portrait orientation, and edge-to-edge.
- UI code mixes React Native layout primitives with utility `className` styling; follow the existing pattern in `app/(tabs)/index.tsx` and `src/components/SmartInputSheet.tsx` rather than introducing a new styling system.
- The app is phone-first and tab-based; the current tab set is Home, Analytics, Wallets, Map, and Profile.
- Android permissions and window behavior are declared in `android/app/src/main/AndroidManifest.xml`; keep location, biometrics, storage, and splash/theme wiring consistent with the current manifest.
- Useful commands from the project root:

```powershell
npm run start
npm run android
npm run web
cd android; .\gradlew.bat assembleDebug
cd android; .\gradlew.bat installDebug
cd android; .\gradlew.bat clean
```
